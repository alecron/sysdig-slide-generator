How It Works

Five phases, fully automated

The skill runs a five-phase pipeline. Let me walk you through each one.

The 5-Phase Workflow

From references to a polished deck in minutes

  • Phase 1: Input Gathering — Topic, audience, slide count, and optional reference materials
    • Just describe what you want — the skill asks for anything missing
  • Phase 2: Reference Analysis — Parallel AI agents analyze each source document
    • PDFs, transcripts, markdown — all processed simultaneously
  • Phase 3: Content Synthesis — Merges analyses into a narrative arc
    • Generates a slide-by-slide outline for your approval
  • Phase 4: HTML Generation — Assembles the reveal.js presentation
    • Selects layout types, fills content, adds speaker notes
  • Phase 5: Output — Writes presentation.html, speaker-notes.md, and outline.md
Phase 1 is just a conversation — tell Claude what you need. Phase 2 is where the magic happens: if you provide reference documents, the skill launches parallel AI agents that each analyze a different source file, extracting key points, quotes, and statistics. Phase 3 synthesizes all that into a narrative structure and shows you an outline before generating anything. Phase 4 builds the actual HTML using our layout templates. Phase 5 delivers the files and opens the presentation in your browser.

Parallel Reference Analysis

What the Agents Extract

  • Key Points — The 3-5 most important takeaways
    • Filtered for relevance to your topic and audience
  • Notable Quotes — Impactful statements with attribution
    • Ready to drop into quote slide layouts
  • Data & Statistics — Numbers that support the narrative
    • Formatted for callout and featured box slides
  • Narrative Threads — How each source connects to the story

Why Parallel?

Each reference document gets its own AI agent running simultaneously. A deck with 10 source files doesn't take 10x longer — all analyses happen at once.

Result: faster generation, deeper analysis per source.

This is the key architectural decision. Instead of processing documents sequentially, each source gets its own subagent. They all run in parallel, extracting structured information that feeds into the synthesis phase. This means you can throw 10 PDFs, 5 transcripts, and a dozen markdown files at it, and it processes them all simultaneously.

Three Outputs, One Command

presentation.html

Self-contained reveal.js deck. Open in any browser — no server, no dependencies. Includes embedded CSS theme, SVG decorations, and speaker notes.

speaker-notes.md

Full presenter script in markdown. 2-4 sentences per slide in a conversational tone. Use it as-is or as a starting point for your own delivery.

outline.md

Slide-by-slide structure overview. Layout type, title, content summary, and notes for each slide. Approved before generation begins.

You get three files. The HTML is completely self-contained — no Node.js, no npm, no local server. Just open the file. The speaker notes give you a full script you can rehearse from. And the outline is generated first so you can approve the structure before the skill builds the full deck.

23 Layout Types

Every slide has a purpose

Let's look at the visual toolkit. The skill has 23 distinct layout types, each designed for a specific type of content.

Layout Categories

Structural Slides

  • Cover (Lumin, Dark, Clock)
    • 3 variants for different opening styles
  • Agenda
    • Numbered list on lumin background
  • Section Dividers
    • 4 SVG patterns: chevrons, diamonds, mountains, gradient
  • Closing
    • Logo + tagline for Q&A

Content Slides

  • One Column (Light & Dark)
    • Standard bullet slides
  • Two & Three Column
    • Comparisons, triads, side-by-side
  • Content + Featured / Quote
    • Split layouts with emphasis cards
  • Tables, Callouts, Full Quotes
    • Data, stats, and testimonials
The layouts fall into two groups. Structural slides create rhythm — they're the covers, dividers, and closings that frame your story. Content slides carry the information. The skill automatically varies between these to keep the audience engaged. The rule is: never more than 3 of the same layout in a row, and aim for a 60/40 split between content and visual slides.

Getting Started

Install in 30 seconds

Let's get you up and running. Installation takes two commands.

Installation

As a Claude Code Plugin

  • Add the marketplace
    • /plugin marketplace add alecron/sysdig-slide-generator
  • Install the plugin
    • /plugin install sysdig-slide-generator

As a Local Skill

  • Clone and symlink
    • git clone the repo, then symlink into ~/.claude/skills/

I described what I needed, pointed it at my reference folder, and had a polished 16-slide deck in minutes. The speaker notes alone saved me an hour.
